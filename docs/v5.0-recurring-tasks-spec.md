## 3. Yeni Model

### 3.1 `TaskOccurrence` (TaskCompletion yerine geçer)

```python
class TaskOccurrence(db.Model):
    """Rutin görevin BİR periyotluk tamamlanma kaydı.

    period_key periyod tipine göre kanonik string format:
      - Günlük:    "2026-05-04"      (ISO date)
      - Haftalık:  "2026-W18"        (ISO week)
      - Aylık:     "2026-05"         (year-month)
      - Yıllık:    "2026"            (year)

    UNIQUE(task_id, period_key) — bir görev bir periyotta en fazla bir kez
    tamamlanır.
    """
    __tablename__ = "task_occurrences"
    id           = db.Column(db.Integer, primary_key=True)
    task_id      = db.Column(db.Integer, db.ForeignKey("tasks.id", ondelete="CASCADE"), nullable=False)
    period_key   = db.Column(db.String(20), nullable=False)
    completed_at = db.Column(db.DateTime, default=datetime.utcnow)
    completed_by = db.Column(db.Integer, db.ForeignKey("users.id"), nullable=True)
    __table_args__ = (db.UniqueConstraint("task_id", "period_key", name="uq_task_period"),
                      db.Index("ix_occurrence_period", "period_key"))
```

### 3.2 Helpers (`models/database.py`)

```python
def _period_key(period: str, dt) -> str | None:
    """Periyot ve tarih için kanonik key — Tek Seferlik için None."""
    if period == "Günlük":
        return dt.isoformat()
    if period == "Haftalık":
        iso = dt.isocalendar()
        return f"{iso.year:04d}-W{iso.week:02d}"
    if period == "Aylık":
        return f"{dt.year:04d}-{dt.month:02d}"
    if period == "Yıllık":
        return f"{dt.year:04d}"
    return None  # Tek Seferlik
```

### 3.3 `Task` instance method'ları

```python
def is_done_now(self, today=None) -> bool:
    """Rutin için: bu periyodun TaskOccurrence kaydı var mı?
    Tek Seferlik / proje / diğer için: self.is_done flag'i."""
    if self.category == "routine" and self.period != "Tek Seferlik":
        key = _period_key(self.period, today or date.today())
        if not key:
            return False
        return TaskOccurrence.query.filter_by(task_id=self.id, period_key=key).first() is not None
    return bool(self.is_done)

def is_overdue_now(self, today=None) -> bool:
    """Rutin overdue: bu periyot için completion yok VE periyot bitti.
    Diğer kategoriler için: deadline geçti VE açık."""
    today = today or date.today()
    if self.category == "routine" and self.period != "Tek Seferlik":
        # Bu periyot bitti mi VE completion yok mu?
        # Günlük: dün gece bitti
        # Haftalık: geçen pazar gece bitti
        # Aylık: geçen ay sonu bitti
        # Yıllık: geçen yıl sonu bitti
        # Implementation: previous period_key için completion bak
        prev_key = _previous_period_key(self.period, today)
        if not prev_key:
            return False
        return not TaskOccurrence.query.filter_by(task_id=self.id, period_key=prev_key).first()
    return (not self.is_done) and self.deadline and self.deadline < today
```

## 4. Migration

**Bir kerelik veri taşıma — `init_db()` içinde:**

```python
# v5.0 Migration — TaskCompletion → TaskOccurrence
inspector = inspect(db.engine)
if "task_completions" in inspector.get_table_names() and "task_occurrences" not in inspector.get_table_names():
    db.create_all()  # task_occurrences tablosunu oluştur
    # Eski veriyi taşı: year+month → period_key "YYYY-MM"
    db.session.execute(text("""
        INSERT INTO task_occurrences (task_id, period_key, completed_at, completed_by)
        SELECT task_id,
               printf('%04d-%02d', year, month),
               completed_at,
               completed_by
        FROM task_completions
    """))
    db.session.commit()
    print(f"v5.0 Migration: {db.session.query(TaskOccurrence).count()} TaskOccurrence taşındı")
    # task_completions tablosu DROP edilmez — rollback için kalır.
    # Bir sonraki major'da temizlenir.
```

**Önemli:** SQLite DDL transaction-safe değil, dikkatli ol. PostgreSQL prod'da migration script'i ayrı tutulabilir.

## 6. APScheduler Etkisi

`services/notifier.py` mevcut. **Yeni job eklemiyoruz** çünkü TaskOccurrence yokluğunda görev otomatik "açık" görünür — reset gerekmez. Sadece bildirim akışı `is_overdue_now()` helper'ını kullanır.

**Opsiyonel:** Cron her gün 00:05 → her rutin için `last_notified` reset (yeni periyot başladıysa). Bu yapılırsa kullanıcı her periyot başında bildirim alır. **Levent kararı (Soru 4)**.
